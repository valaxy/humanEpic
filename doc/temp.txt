# GDScript代码规范
- 总是使用godot4.X最新版本API和语法
- 枚举类型定义在单独的文件`<EnumName>.enum.gd`中
- 语法细节：
  - `Dictionary`使用`key = value`的形式书写
  - `_init()`函数的参数不要带下划线`_`或`p_`前缀，通过`self.pos = pos`的形式赋值，而不是`pos = _pos`的形式
  - 拒绝null值检查过载：如果成员在初始化函数中存在，则不必进行null检查
- 注释：
  - 文件注释：文件开头用文档注释（使用`##`语法）说明当前文件的主要功能，稍微详细一点，便于AI和人类理解
  - 函数注释：对非godot内置函数尽量添加一行文档注释说明函数功能
  - 类属性注释：对类的公开属性，在行末添加一段文档注释进行描述，简洁一点
  - 信号注释：在行末添加一段文档注释说明信号的用途，简洁一点

# 大世界

- **地理系统** (src/geography): 管理地形、覆盖物规则及区域逻辑。
- **经济系统** (src/economy): 管理资源产出、存储及贸易循环。
- **军事系统** (src/unit): 管理单位行为、寻路、战斗结算及领土占据。
- **显示层级系统** (src/layer): 负责战略/细节视角的动态渲染优化。
- **持久化系统** (src/persistence): 处理地图及进度的存档与加载。·
